Desktop

Existing desktop environments are extremely complex in terms of binary size, memory usage and especially development APIs. Of course, some of the complexity is warranted, as modern desktops serve a variety of different users and functions. But how much is needed and how much is needless weight?

This desktop project is testing the limits of simplicity. Drawing upon the best libraries around, this desktop includes a web browser, e-mail client, instant messenger, media player, terminal and calculator, all with internationalization support... in 1573 lines of code! :)

The desktop is cross-platform (except for the terminal, which requires a native shell), and on UNIX requires only an X display server to run. The current code is alpha, but rapidly progressing.

Besides Java's built-in APIs, the system uses JAIM for Instant Messenger connectivity to AOL's TOC network, and will soon incorporate JRex, a Java wrapper for embedding Mozilla's GECKO rendering engine.
